# PDF 合併工具（純前端、免上傳）

一個 **純前端、可在 GitHub Pages 執行的 PDF 合併工具**。  
支援多個 PDF 拖曳排序、**選擇頁碼範圍合併（如 1-3,5,7-99）**、  
並即時以 **縮圖預覽選取的頁面**，所有處理都在瀏覽器中完成，PDF 不會上傳到任何伺服器。

👉 線上示範  
https://suwaillya.github.io/pdf-merge/

---

## ✨ 功能特色

### 📂 多 PDF 合併
- 支援一次加入多個 PDF
- 加入後仍可繼續追加檔案
- 拖曳調整合併順序（桌機 / 手機皆可）

### 🔢 選擇頁碼範圍（進階）
- 每個 PDF 可單獨指定要合併的頁面
- 支援格式：
- 留空表示「全部頁面」
- 1 秒內未繼續輸入即自動解析
- 格式錯誤會即時提示，不影響輸入體驗

### 🖼️ PDF 頁面縮圖預覽
- 點開 PDF 即可看到「實際會被合併的頁面」
- 縮圖會隨頁碼選擇即時更新
- 為避免效能問題，預設最多預覽前 200 頁（可自行調整）

### 🗑️ 單檔移除
- 可隨時移除某個 PDF，不影響其他檔案

### 🔐 隱私與安全
- **純前端處理**
- 不上傳、不儲存、不追蹤
- 適合處理敏感文件

---

## 🧠 為什麼做這個工具？

市面上的 PDF 合併服務多半需要：
- 上傳檔案到伺服器
- 登入帳號
- 或限制檔案大小與次數

這個專案的目標是提供一個：
- 簡單
- 透明
- 可自行部署
- 不依賴後端

的 PDF 合併解決方案，讓使用者可以完全掌控自己的檔案。

## 🛠️ 使用技術

- HTML / CSS / Vanilla JavaScript
- [pdf-lib](https://github.com/Hopding/pdf-lib)（PDF 合併）
- [pdf.js](https://github.com/mozilla/pdf.js)（頁面縮圖預覽）
- [SortableJS](https://github.com/SortableJS/Sortable)（拖曳排序）
- GitHub Pages（靜態部署）
